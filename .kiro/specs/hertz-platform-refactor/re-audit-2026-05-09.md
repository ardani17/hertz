## Kesimpulan Tingkat Tinggi

Implementasi sudah cukup kuat untuk shell desktop, route publik, seed, landing, Blog/Outlook/Tools wrapper, membership auth dasar, dan sebagian aksi sosial.

Namun implementasi belum 100% karena domain HERTZ baru belum benar-benar menjadi sumber utama. Banyak route HERTZ masih memakai `FeedService`, `FeedRepository`, `feed_posts`, `post_*`, dan `community_notes` lama. Tabel `hertz_*` sudah ada dan seed terisi, tetapi sebagian besar feed/composer/interaction belum menulis atau membaca dari tabel `hertz_posts`, `hertz_post_media`, `hertz_reactions`, `hertz_comments`, dan `hertz_community_notes`.

## Findings Utama

### P0 - HERTZ post API/detail belum sehat di runtime lokal

- `GET /api/hertz/posts` mengembalikan 500.
- `/hertz/post/hz_live01` mengembalikan 404.
- Dampak: Requirement 2.3, 8.1, 8.10, 20.12 belum bisa disebut selesai penuh.
- Catatan: build tetap lulus, jadi ini bukan TypeScript error, melainkan runtime/data/env/service path.

### P0 - Domain `hertz_*` belum dipakai penuh untuk feed

- `shared/repositories/hertzPostRepository.ts` hanya `extends FeedRepository`.
- `shared/services/hertzPostService.ts` hanya export alias `FeedService`.
- `frontend/src/app/api/hertz/posts/*` masih instantiate `FeedService`.
- `FeedRepository` masih insert/select `feed_posts`.
- Dampak: Requirement 8.9 dan 16.1-16.14 baru terpenuhi di schema/seed, belum sebagai runtime domain utama.

### P1 - Category domain masih campuran lama

- Spec meminta `Trading_Room`, `Life_Coffee`, `General`, `Community_Note`.
- Runtime service masih validasi kategori lama: `trading`, `life_story`, `general`.
- Migration `hertz_posts` sudah punya kategori baru: `trading_room`, `life_coffee`, `general`, `community_note`.
- Dampak: UI label terlihat benar, tetapi API/domain belum konsisten.

### P1 - Composer media belum menyimpan ke `hertz_post_media`

- Web composer memakai `mediaIds` lalu `attachMediaToArticle`.
- Feed media masih dibaca dari `media.article_id`.
- `hertz_post_media` sudah ada dan di-seed, tetapi bukan jalur utama composer.
- Dampak: Requirement 8.8-8.9 belum penuh.

### P1 - DM masih phase minimal, belum sesuai spec lengkap

- Ada inbox, create conversation, thread, send, mark read, dan polling 7 detik.
- Belum terlihat endpoint:
  - user search modal sesuai `/api/dm/users/search` atau padanan HERTZ lengkap,
  - block/unblock API,
  - report API,
  - archive API,
  - delete message API,
  - attachment upload/send validation 4 images/5MB,
  - unread/filter Admin/Archived UI yang nyata.
- UI masih menampilkan `Conversation {id.slice}` tanpa peer profile, unread count, filters, atau attachment picker.
- Dampak: Requirement 14 banyak yang parsial.

### P1 - Blog member flow masih create-only

- `POST /api/blog` ada dan publish langsung.
- Belum terlihat API member untuk edit/delete own blog.
- Blog cover image dan SEO per artikel belum kuat pada create API.
- Report/takedown guardrail member-facing belum jelas.
- Dampak: Requirement 11.6-11.12 parsial.

### P1 - Admin HERTZ ada, tetapi service masih bernama lama dan coverage belum penuh

- Route `/admin/hertz` dan API `/api/admin/hertz/*` ada.
- Shared service `signalLedgerAdminService.ts` masih ada dan dipakai sebagai admin backend.
- Blog takedown, DM report limited-context review, dan moderation report tools lintas Blog/post/note/DM belum terbukti lengkap.
- Dampak: Requirement 15 parsial.

### P1 - Docker QA belum benar-benar sehat dari config langsung

- `docker compose config` lulus secara struktur tetapi env kosong menghasilkan `DATABASE_URL: postgresql://:@db:5432/`.
- Ini mungkin karena deploy script yang benar melakukan `source .env` dan export, tetapi command compose langsung belum membuktikan runtime production path sehat.
- Dampak: Requirement 20.11 belum 100% terbukti.

### P2 - Test coverage HERTZ masih wiring-heavy

- Test lulus 638, tetapi HERTZ-specific integration test banyak mengecek keberadaan file/string, bukan perilaku DB/API end-to-end.
- Nama test lama `signalLedger*` masih ada.
- Tidak ada bukti test aktual untuk DM attachment, block/report, archive, hertz_post_media runtime, atau hertz_posts sebagai source of truth.
- Dampak: Requirement 20.9 dan task 12.1 terlalu optimistis.

### P2 - Stale Gallery masih ada sebagai route lama

- `/gallery`, `/api/gallery`, component gallery, sitemap gallery masih ada.
- Ini tidak muncul di left rail HERTZ final, jadi tidak memblokir menu final.
- Namun jika spec dimaknai "hapus stale Gallery dari produk final", maka route lama masih tersedia.

## Rekomendasi Urutan Perbaikan

1. Jadikan `hertz_posts` source of truth untuk `/api/hertz/posts`, `/hertz`, dan `/hertz/post/[shortId]`.
2. Migrasikan interaction runtime ke `hertz_reactions`, `hertz_comments`, `hertz_reposts`, `hertz_bookmarks`, `hertz_community_notes`.
3. Samakan kategori API dengan domain baru: `trading_room`, `life_coffee`, `general`, `community_note`.
4. Pindahkan composer media ke `hertz_post_media`.
5. Lengkapi DM backend/UI: search member, unread count/filter, archive, delete, block, report, attachments.
6. Lengkapi Blog edit/delete/cover/SEO/report/takedown.
7. Perkuat admin HERTZ moderation untuk Blog, reports, DM reported context.
8. Tambahkan tests perilaku nyata, bukan hanya file/string existence.
9. Jalankan Docker via `bash deploy-docker.sh` dengan `.env` lengkap, bukan hanya `docker compose config`.
10. Ulang visual QA terhadap Figma frames setelah runtime domain sehat.
